Q: 有处理过点击穿透相关的问题？

A：有处理过，点击穿透的话，在以前比较会出现，其实他的原理就是click事件触发的时候，他会有一个300毫秒得延迟，这样子把两个元素叠在一起的时候，就一个元素覆盖在另一个元素上，然后你点击元素，元素隐藏的时候，就会因为300毫秒的延迟，就会影响到下面元素的一个click事件，然后其实在现在的话，移动端300ms延迟的问题已经去掉了的话，就不会这样的。

Q：请将“setTimeout”封装成一个Promise，5秒后打印”hello，world“

A:

/* Q：请将“setTimeout”封装成一个Promise，5秒后打印”hello，world“ */

function sleep(timeout){
  return new Promise((resolve, reject) => {
    setTimeout(resolve, timeout * 1000)
  })
}

口述深拷贝：

深拷贝，主要做了递归调用的方式，可能先判断它是否为空，或者是普通的数据类型，如果不是object，就可能直接就被他们祛除了，，如果是object，或者数组，就直接递归调用这个克隆方法，然后把当前值穿进去，再去遍历object对象，然后在return出去，大概就是这样一个逻辑

Q：wx小程序为什么不用原生的，用的Uniapp？

A：可能也考虑到，2点，第一点，公司架构的话，前端用的vue写的，为了能够统一这些代码的一个通用性，所以就选择了一套unipp，原生写比较难收，写四个文件是一个页面，uniapp还可以写成单文件，语法类vue。

Q：uniapp影响深刻哪些坑？

A：api和微信保持一致，微信一些特有的配置，微信官网api频繁更新，uniapp这边同步更新api有一定时间的延迟，增加或者变动，然后这些uniapp文档跟不上，你就不太能够找得到这些配置怎么配上去，坑坑就习惯了。

Q：项目流程说一下？

A：项目确立之后，就可能会安排前后端的人手，然后会去开个启动会议，介绍整个项目背景，开发周期，人员安排，还有一些整个我们这个项目的，把我们项目管理的项目通过任务管理器，trellow去把他分解出来，分解出来，按照周期排好，可能比如说这一周完成什么。有bug，然后我们再去修复哪些反馈，到整个项目结束。gitflow标准的流程，然后把自己的任务在开发分支提交，然后其你去合并到我们的开发分支，有代码review

Q：怎么review？

A：一般前端review前端，后端review后端。

Q：merge模式，git rebase，？

A: 合并代码用的merge，处理冲突用的rebase。 因为merge会新增加一个commit，记录，rebase把这条开发分支提前到当前分支。
