# 02-promiseapi使用

# promiseapi的说明

    /* 
    Promise构造： new Promise(excutor)
    1. excutor函数：执行器(resolve, reject) => {}

    
    //xx.prototype.xxx() 只有实例对象才能调用 pro.then(),而不是Promise.then()
    
    2. Promise.prototype.then() 返回成功或者失败的新的promise

    3. Promise.prototype.catch() 返回失败的promise

    4. Promise.resolve() 返回一个成功或者失败promise对象

    5. Promise.reject() 返回成功或者失败promise对象

    6. Promise.all方法：(promises) => {}
      promises: 包含n个promise的数组
      说明:返回一个新的promise，只有所有的promise都成功才成功，有一个失败直接失败
    
    7. Promise.race (promises) => {}
      promises: 包含n个promise的数组
      说明：返回一个新的promise，第一个完成的promise的结果状态就是最终的结果状态
    */

# promise关键问题们

    /* 
    1. 如何改变promise状态？
      1.1 resolve（value）如果当前是pendding就变为resolved
      1.2 reject（reason）如果当前是pendding就变为rejected
      1.3 抛异常 如果当前是pendding就会变为rejected

    2.一个promise指定多个成功/失败回调函数，都会调用吗？
      当promise改变为对应状态时都会调用 
    */
   const p = new Promise((resolve, reject) => {
      // resolve(1) // promise变为resolved成功
      // reject(2) // promise变为rejected失败
      // throw
      // throw new Error('出错了') // promise变为rejected失败
      // 不应定抛出error，可以跑任意
      throw 3 // 抛出什么就是reject什么 reject 3
   })
   p.then(value => console.log( value ), reason => console.log( 'reason',reason ))
   p.then(value => console.log( value ), reason => console.log( 'reason2',reason ))
   // 多个调用promise，全部都会执行,是可以在两个失败监听里去做不同的事情

    /* 
    3. 改变promise的状态和指定回调函数谁先谁后？
      1. 都有可能，正常情况下是现制定回调再改变状态，但是也可以先改变状态再指定回调
      2. 如何先改状态再指定回调
        2.1 在执行器中直接调用resolve和reject
        2.2 延迟更长时间才调用then
      3. 什么时候才能得到数据
        3.1 如果先指定回调，当状态改变，回调就调用，得到数据
        3.2 如果先改变状态，当指定回调时，回调函数调用，得到数据
    */
  //  常规：先指定回调函数，后改变的状态
  //  如果先指定回调，当状态改变，回调就调用，得到数据

  //     setTimeout(() => {
  //       resolve(1) // 后改状态，同时指定数据,异步执行回调函数
  //     }, 1000)
  //  })
  //  p.then(// 先指定回调，保存当前指定的回调函数
  //    value => {},
  //    reason => {console.log( reason )}

  // //  如果先改状态
  // const p = new Promise((resolve, reject) => {

  //    reason => {console.log( 'reason2',reason )}
  //  )


  // 实现先改变状态和数据，后指定回调
   const p = new Promise((resolve, reject) => {
     setTimeout(() => {
      resolve(1) // 先改状态，同时指定数据
     }, 1000)
   })
   setTimeout(() => {
    p.then((value) => {
      console.log( value )

# promise关键问题3

    /* 
    4. promise.then返回的新promise的结果状态由什么决定？
      1. 简单表达：由then()指定的回调函数执行的结果决定
      2. 详细表达
        2.1 如果抛出异常，新promise变为rejected，reason为抛出的异常
        2.2 如果返回的非promise的任意值，xinpromise变为resolved，value为返回的值
        2.3 如果返回的是一个新promise，次promise的结果就会成为新promise的结果
    */
   new Promise((resolve, reject) => {
     resolve(1)
    // reject(1)
   }).then(
     value => {
       console.log( 'onResolved1()', value )// 1
      //  return 2 // 这个就是下一个then的结果
      // return Promise.resolve(3) // 由then()指定的回调函数执行的结果决定
      // return Promise.reject(4)// 由then()指定的回调函数执行的结果决定
      throw 5 // 等价于reject 5
     },
     reason => {
       console.log( 'onRejected1()', reason )
     }
   ).then(
    value => {
       console.log( 'onResolved2()', value )// undefined
     },
     reason => {
       console.log( 'onRejected2()', reason )
     }
   )

    /* 
    5. promise如何串联多个操作任务
     5.1 .then可以返回一个新的promise，可以开诚then的链式调用
     5.2 通过then的链式调用串联多个同步/异步任务
    */
    new Promise((resolve, reject) => {
     setTimeout(() => {
       console.log( '执行异步任务1' )
       resolve(1)
     }, 1000)
   }).then(
     value => {
       console.log( '任务1的结果为', value )
         return new Promise((resolve, reject) => {
            //  如果一旦是异步的必须包装在Promise里
              //  启动异步任务3
          setTimeout(() => {
            console.log( '执行异步任务3' )
            resolve(3)
          }, 1000)
        })
     }).then(
    value => {
       console.log( '任务2的结果为()', value )
     }
   )
