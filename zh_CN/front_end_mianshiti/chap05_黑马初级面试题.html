# h5c3问题：cookies,sessionStorage,localStorage的区别?

  • 分析：考察求职者对本地存储的方式是否熟悉，因为初学者会把cookies，sessionStorage，localStorage搞混！！！

  • 解答：

    • cookies: 他是网站用来标记用户身份的一段数据,一般情况下是一段加密的字符串，并且默认情况下只会在同源的http请求中携带
    • sessionStorage: 他是浏览器本地存储的一种方式，以键值对形式进行存储，并且存储的数据会在浏览器关闭之后删除
    • localStorage：他也是浏览器本地存储的一种方式，和sessionStorage一样，也是以键值对的形式存在的，不过他存储的是本地持久化的数据，一般不主动删除，数据会一直存在

# h5c3问题：display:none;与visibility:hidden;的区别？

  • 分析: 这是一个易出错的css问题，没有大量的实际开发经验的程序猿，很难了解他们直接的区别

  • 解答：

    • 共同点：他们都可以让元素不可见
    • 不同点：
      • display: none;会让元素完全从渲染树种消失，不渲染的时候不占据任何空间。而visibility:hidden;不会让元素从渲染树中消失，不渲染时元素继续占领空间，只是内容不可见
      • display: none;是非继承属性，子孙节点消失是因为元素从渲染树中消失造成，通过修改子孙节点属性无法显示，而visibility:hidden;是继承属性，子孙节点消失由于继承了hidden，通过设置visibility:hidden;可以让子孙节点显式

# h5c3问题：为什么要初始化CSS样式?

  • 分析：这个问题主要是看你有没有实际开发经验，因为对于没有实际开发的同学是不知道正式项目是需要初始化CSS样式的，也就是清理掉一些元素默认所携带的样式，比如默认的padding,margin

  • 解答：因为浏览器有兼容问题,不同浏览器对有些元素的默认样式是不同的，并且默认的样式会影响到项目最终的布局，如果没有对CSS初始化，那么有可能会出现浏览器之间的页面显示差异

# h5c3问题：什么是postcss，postcss有什么作用？

  • 分析：有很多工作了两年，三年的开发者依然无法解释清楚postcss是什么，以及他的作用

  • 解答：

    • 首先明确 postcss 是一个平台
    • 基于这个平台，可以使用一些插件，来优化css的代码，比如说，autoprefixer插件，他就需要基于postcss使用，作用是可以帮助我们css增加上不同的浏览器厂商前缀

# js问题：解释一下闭包？以及闭包解决了什么问题？

  • 分析：闭包一直是前端非常关注的问题，老问题了

  • 解答：

    • 什么是闭包？闭包就是能够读取其他函数内部变量的一个函数，通常情况下，我们会在一个函数中去创建另外一个函数，然后通过新创建的这个函数来访问上层函数的局部变量，被访问到的局部变量会始终保持在内存之中

# js问题：原型和原型链分别是什么？有什么特点？

  • 分析：原型和原型链的问题和闭包一样，也是面试被问到频率很高的问题

  • 解答:

    • 每一个函数都有一个prototype属性，这个属性会指向一个对象，就是通过调用该函数的构造函数而创建的实例的原型，可以通过实例对象的 proto 来访问到这个原型对象。
    • 实例在创建的时候，就会默认关联原型，并且会从原型之中继承到一些属性。
    • 而每一个原型对象都会存在一个constructor属性，这个属性会指向关联的这个构造函数。
    • 当访问一个实例对象的属性时，如果说这个实例对象中没有属性，那么js引擎就会去该实例对象的原型对象中去找
    • 原型链: 如果属性在原型对象中也找不到，那么就通过__proto__回去原型的上一层原型去找，一直找到最上层的原型（也就是找到Object.prototype为止,知道找到的原型是一个null为止）

# js问题：谈谈对this对象的理解

  • 分析：在js中一直存在一个本不应该存在的问题，那就是关于this指向的问题，关于this在其他语言中是非常明确地（但是js因为历史原因，导致this的指向一致是重难点）

  • 解答：

    • this总是指向**函数的直接调用者，而非间接调用者
    • 如果有new关键字，this指向new出来的这个对象
    • 在event中，this指向触发事件的这个对象，特殊的是，ie中的attachEvent中的this总是指向全局对象:Window

# js问题：谈谈你对webpack的看法？

  • 分析: 对于中级工程师来说，webpack的内容一般不会问的很深，因为除非是高软或者架构，否则一般不会配置webpack，但是这并不是你不会的理由

  • 解答

    • webpack是一个js模块打包工具，可以使用webpack管理项目中的js模块依赖。
    • webpack提供了默认了配置，比如说devServer，我们可以利用devServer来快速启动一个开发时的web服务器。
    • 因为webpack默认只能打包js文件，所以webpack额外提供了loader的概念，我们可以使用loader来预处理一些文件，并且可以打包除了js之外的任何静态资源文件。
    • 另外webpack还提供了插件的概念，我们可以使用很多的插件来在webpack上做一些辅助性的工作，比如HtmlWebpackPlugin这种用来创建html文件的插件。

# js问题：谈谈你对promise的了解?

  • 分析：promise是开发中经常用的一个点，必须要掌握

  • 解答：

    • 首先promise是es6提出的一个异步编程解决方案，相比于传统的容易陷入回调地狱的异步回调方案来说，Promise会让异步的操作变得更优雅。
    • es6规定promise是一个构造函数，所以我们需要通过new关键字来生成一个promise的实例对象。
    • promise的构造函数接受一个函数作为参数，函数的代码在new Promise的时候会立即执行，我们可以在这里去执行异步代码的操作。并且该函数默认存在两个参数分别是resolve，reject，这两个参数也是函数，用来标记异步执行的状态。
    • 比如resolve，当promise的异步操作完成的时候，可以调用resolve函数，来标记当前异步操作已经完成了。
    • 而reject，是在异步操作失败的时候调用，用来标记当前的异步操作已经失败了。
    • 这些标记的状态我们可以通过promise实例对象的.then方法和.catch方法去接收，其中.then方法是异步完成的回调，.catch是异步失败的回调。

# js问题：null和undefined的区别是什么？

  • 分析：一般在笔试

  • 解答：

    • undefined： 表示不存在这个值，他是一个变量最原始的状态 var b; // undefined
    • null：他是具体的一个值，只不过这个值是一个空值而已。 var b = null; // null

# js问题：什么是同步？什么是异步？区别是什么？

  • 分析：对于任何编程语言来说，同步和异步都是重点

  • 解答：

    • 同步：线程被阻塞，等待任务返回结果。
    • 异步：异步就是线程不会被阻塞，任务完成通知js引擎。

# js问题：什么是eventloop？

  • 分析：主要是询问js线程机制，因为你只有了解js的单线程机制，你才可以理解什么是eventloop。

  • 解答：

    • js语言一大特点就是单线程，也就是说，同一时间只能做一件事。
    • 单线程就意味着，所有的任务需要排队，前一个任务结束，后一个任务才可以执行，如果前一个任务耗时很长，后一个任务就不得不等待着。
    • 于是，所有的任务可以分成两种，一种是同步任务（synchronous），另一种是异步任务（asynchronous）。
    • 同步任务：在主线程排队执行的任务，只有前一个任务执行完毕，才可以执行后一个任务。
    • 异步任务：不进入主线程，而进入“任务队列”（tack queue）的任务，只有"任务队列"通知主线程，某个异步任务可以执行了，该任务才会进入主线程执行。
    • 而当主线程中的任务执行完成，主线程从“任务队列”中读取事件，这个过程是循环不断的，这就是event loop（事件循环）。

# js问题：bind，call，apply的区别?

  • 分析：这三个方法在react项目用的多，会牵扯到this指向,问道的概率大

  • 解答：

    • call和apply都是为了解决改变 this 的指向，作用都是相同的，只不过参数不同。
    • 除了第一个参数外，call方法可以接收一个参数列表，apply方法只可以接收一个参数数组
    • bind和call几乎一致，但是这个方法只是返回一个函数，并且我们可以通过bind实现柯里化。

  • 代码

let a = {
  value: 1
}
function getValue(name, age) {
  console.log(name)
  console.log(age)
  console.log(this.value)
}
getValue.call(a, 'z9', '22') // z9, 22, 1
getValue.apply(a, ['z9', '22'])// z9, 22, 1
let getValueBindKeLiHua = getValue.bind(a, 'z9', 22)
getValueBindKeLiHua()// z9, 22, 1

# js问题：typeof和instanceof的区别？

  • 分析：经常问到，容易搞混。

  • 解答：

    • typeof：会返回一个值的类型，对于基本数据类型 就是两个点加类型（ 'symbol' ），除了null都可以返回直接( object ) ,这是js的一个bug，而对于引用数据类型来说，除了函数之外（console.log // 'function'）,其他都会返回object

# vue问题：请说一下vue实现双向数据绑定的原理？

  • 分析：大概率问

  • 解答：

    • vue实现数据双向绑定主要是：采用数据劫持结合发布-订阅者模式的方式。
    • 数据劫持：通过Object.defineProperty()来劫持对象各个属性的setter，getter。
    • 发布-订阅者模式：在数据变动时发布者发布消息给订阅者，触发相应的监听回调。
    • 总的来说，当把一个普通js对象传给vue实例来作为他的data选项时，Vue将遍历他的属性，用Object.defineProperty（）监听他们的 getter,setter方法，这样，它们就可以让Vue追踪依赖，在对象的属性被访问（get）和修改（set）时通知变化。

# vue问题：请说下Vue中如何实现参数传递？

  • 分析 ： 大概率问

  • 解答：

    • 父组件给子组件传值，子组件通过props属性接收数据
    • 子组件给父组件传值，子组件通过$emit方法传递参数
    • 兄弟组件传值：对于兄弟组件传值来说，兄弟组件之间其实没有任何相互依赖关系，也就是说没有关系，对于没有关系的两个组件，如果想进行数据传递，那么就需要一个第三方，通过第三方来为两个没有关系的组件建立关系，目前比较通用的方式有两种：
    • 一个是 eventBus 方案，建立一个事件中心，通过它传递事件来进行数据传递。
    • 第二个方式就是通过 vuex 来进行数据共享。

# vue问题：路由Vue-router实现：hash模式和history模式？

  • 分析：初中级工程师，就是一个加分项,其实初中级不会让你去部署vue项目的

  • 解答

    • hash模式：是vue-router的默认模式，使用哈希，url中会出现#字符，#以及#之后的字符称为hash（哈希），特点：hash虽然在url中，但不被包括在http请求之中。
    • history模式：国内线上项目使用的比较多的还是history模式，这模式下url中不会存在#这种字符，不过这种模式下，需要与后端程序猿进行配合，因为Vue打包项目通常是一个单页应用，也就是任何的URL其实指向的都是同一个html文件，所以我们要在apache和nginx中配置，当url匹配不到资源的时候，指向同一个html页面的配置。

# vue问题：keep-alive /keep-alive的作用是什么？

  • 分析：keepAlive是构建企业级项目的一个点

  • 解答: keepAlive的作用：keep-alive /keep-alive包裹动态组件时候，会缓存不活动的组件实例，主要用于保留组件状态或者避免重新渲染。
