# 03-冲突的产生


# 冲突的产生
条件1：本地当前编辑的文件已经过时

条件2：从服务器端更新下来的修改和本地的修改在“同文件同行才会冲突”

# 冲突的表现
- 文件内 # 用我么的内容还是用服务器端的内容，需要我们手动解决冲突
	<<<<<<< .mine 
  sklfjklfds_sallyChongTu # 冲突时，我们写的代码
  ======= 
  sklfjklfds_harryChongTu; # 冲突时，服务器端最新的代码
  >>>>>>> .r6
  adkslfjklfds_sally111
  hello_harry222

- 目录内 # 由于发生冲突而产生的三个文件
	-rw-r--r--. 1 root root  24 2月  14 02:20 good.log
  -rw-r--r--. 1 root root 120 2月  14 03:17 hello.txt
  -rw-r--r--. 1 root root  61 2月  14 03:17 hello.txt.mine
  -rw-r--r--. 1 root root  48 2月  14 03:17 hello.txt.r5
  -rw-r--r--. 1 root root  62 2月  14 03:17 hello.txt.r6

# sally 先看看.mine文件, 这是sally修改后的文件
cat hello.txt.mine
sklfjklfds_sallyChongTu
adkslfjklfds_sally111
hello_harry222

# sally 再看看.r5文件
cat hello.txt.r5 # 这是sally修改前的文件
sklfjklfds
adkslfjklfds_sally111
hello_harry222

# sally 再看看.r6文件, 这是harry修改后的文件
cat hello.txt.r6
sklfjklfds_harryChongTu;
adkslfjklfds_sally111
hello_harry222

# xx.mine文件：发生冲突时本地文件内容
# xx.r5(小版本号) 文件：发生冲突前文件内容
# xx.r6(大版本号)：发生冲突时服务器端文件内容

# 04-冲突的解决

# 法1：手动解决
1. cat查看发生冲突时产生的三个文件，然后删除发生冲突时产生的三个文件
2. 删除冲突文件内多余的符号
3. 把文件编辑到满意的状态
4. 提交

# 06-冲突如何尽量避免


实际开发，很难避免冲突发生

1. 减少文件过时能减少冲突,尽可能在修改文件前进行更新操作，尽量在最新版基础上修改内容
2. 尽量减少多人修改同一个文件的可能性
3. 加强团队成员之间的沟通
