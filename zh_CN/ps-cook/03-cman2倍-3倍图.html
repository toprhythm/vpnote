# 视口

  • PC端页面那么大，放在手机上显示不下怎么办？

  • 视口，就是浏览器显示页面的区域，视口分为布局视口，视觉视口，理想视口（我们只用理想视口（苹果公司开创的视口，类似于视网膜屏））

  • 一般移动设备的浏览器默认设置一个布局视口，用于解决早起的pc端页面在手机上显示的问题

  • ios，and，基本都将这个视口分辨率设置为980px，所以pc上的网页大多能在手机上呈现，只不过元素看上去都小，一般默认可以手动缩放网页

# meta视口标签

  • 视口宽度 === 设备宽度
<meta name="viewport" content="width=device-width, user-scalable=no, initial-scale=1.0, maximum=1.0, minimum-scale=1.0">
1
  • 可以注释掉这段代码，在f12的移动端查看，字会特别小，但是解开注释，刷新，子会变得很大很舒服

# 物理像素&物理像素比

  • 物理像素点指的是屏幕显示的最小颗粒，是真实的手机里的物理像素，厂商出厂时就设计好了，例如 iphone6: 750 宽 * 1314 高
  • 我们开发时候的1px不是一定等于一个手机真实物理像素的！！！
  • PC端页面，1px等于1物理像素，但是移动端是因为手机真实像素的不同，来比配1px究竟等于多少物理像素
  • 1px能显示的真实物理像素的个数，成为物理像素比/屏幕像素比

# 再深究物理像素&物理像素比

  • 早起的手机屏幕：确实是1px=1dpr
  • 自从视网膜屏幕出现，可以把更多的物理像素点压缩至一块屏幕里，从而达到更多的分辨率，并提高屏幕显示的细腻程度

# 图片在手机中显示的多倍图

  • 对于一张 50px * 50px 的图片，在手机视网膜屏打开，按照刚才的像素比会放大数倍（至少是ip6的两倍），这样会造成图片的模糊！！！
  • 在标准的viewport 理想视口模式中，使用倍图来提高图片质量，解决在高清设备中的模糊问题。
  • 通常使用二倍图，因为ip678的影响，但事现在还存在3倍，4倍图，这个看实际开发公司的需求，背景图片，注意缩放(background-size)问题

# 图片手机，搞清楚

  • 把普通pc图片，放到手机端，就会无缘无故放大两倍，图片变模糊了，怎么办！！！

  • 我们采取的是 放一个100 * 100的前端开发像素图片，然后手动的把图片缩小为50 * 50 前端开发像素，扔到手机端，又被手机端放大2倍，这个图本身就是100 * 100 ，但是清晰度不会被削弱

  • .html

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    /* 我们需要一个50*50的前端开发像素的图片,直接放到我们的手机ip8就会放大到2倍，100 * 100 就会模糊 */
    /* 我们采取的是 放一个100 * 100的前端开发像素图片，然后手动的把图片缩小为50 * 50 前端开发像素，扔到手机端，又被手机端放大2倍，这个图本身就是100 * 100 ，但是清晰度不会被削弱 */
    /* 我们准备的图片比我们实际需要的大小大2倍，那么这种方式就是2倍图 */
    .qingxi {
      width: 50px;
      height: 50px;
    }
  </style>
</head>
<body>
  模糊的：
  <img src="./images/ap50.jpg" alt="">
  <hr>
  清晰地2倍图:
  <img class="qingxi" src="./images/ap100.jpg" alt="">
</body>
</html>

# 工作中不仅仅用插入的图片，还有背景的图片，背景也是可以缩放为原来的2倍的

  • background-size: 规定背景图像的尺寸;
  • background-size: 背景图片宽度 背景图片高度;
  • .html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    div {
      width: 500px;
      height: 500px;
      border: 2px solid red;
      background: url(./images/dog.jpg) no-repeat;

      /* background-size: 图片宽 图片高; */
      /* background-size: 500px 200px; */

      /* 只写一个参数，肯定是宽度，高度省略了，就等比例缩放 */
      /* background-size: 500px; */

      /* 2. 里面 单位可以跟百分比！！，相对于父盒子来说的*/
      /* background-size: 50% 50%; */

      /* 3. 单位 长度 百分比 cover contain */
      /* 
      cover，把背景图像扩展至足够大，以北京图像完全覆盖背景区域
      contain，把图像扩展至最大尺寸，使得其宽度和高度完全适应内容区域
      */
      /* cover，要完全覆盖div盒子，可能有部分背景图片显示不全 */
      /* background-size: cover; */

      /* 宽度拉满，盒子高度不再缩放,可能有部分空白区域 */
      background-size: contain;
    }
  </style>
</head>
<body>
  <div></div>
</body>
</html>

# 背景图的2倍图

  • .html
<!DOCTYPE html>
<html lang="en">

  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    /* 1. 50 * 50 的前端开发像素盒子需要一个背景图片，这个图片还是ip6，还是2倍，100 * 100 的真实像素的图片 */
    /* 2. 我们需要把这个图片缩放一半，也就是50 * 50，background-size */
    div {
      width: 50px;
      height: 50px;
      border: 1px solid red;
      background: url(./images/ap100.jpg) no-repeat;

      background-size: 50px 50px;
    }
    /* 有的时候2倍，有的时候3倍，比较麻烦，怎么办？ */
    /* 使用cman！！！ */
  </style>
</head>

  <div></div>

</html>

2）选择 ios，或者and，选择@2（2倍），选择@3，三倍，选择@1，1倍
