# 01-git是什么?

git: 分布式 版本控制 软件

第一代：文件 拷贝，手动标记版本v1,v2,v3
第二代：本地版本库 
第三代：集中式（A电脑，B电脑， 版本控制中心） 知名软件：SVN
第四代：分布式 A电脑，B电脑，本地服务器，中心服务器，历史版本v1,v2,v3

分布式的好处：服务器挂了，本地还有版本控制，保证代码更完整,保全

# 02-为什么要做版本控制?

当前： UI小姐姐 - 公司总经理 - 老板网红Alex

一年前： UI小姐姐 - 网红Alex 直接对接，小姐姐抓住alex的点位造图

当前：UI小姐姐准备两个版本的样式项目 - 一套总经理满意，alex不满意，一套alex满意总经理不满意，先把总经理满意的交上去，被退回再把alex喜欢的交上去，然后OK了，这就是两个版本的好处

随着项目越来越大，例如 Vue.js 2.5.1, 2.5.2, 2.5.3, ..., 2.6.0

好处：最新的版本出Bug的时候，回滚到前一个版本，很爽

# 05-commit之前必须做

  1. 在commit之前，必须进行个人信息配置：用户名，邮箱,我们电脑上之前装过git，所以运行commit不报错,这个配置只需最开始装机执行一次即可
运行下面这两个命令:

	git config --global user.email "you@example.com" ##可以随便写
	git config --global user.name "you name" ##可以随便写

# 06-git三大区域

git三大区域:
	工作区
		已管理池子
			index.html
		未管理池子
			README.html
	缓存区 git add 把工作取未管理池子的文件提交到暂存区,(是一个缓冲的地方)
	版本库 git commit 把已经在缓存区中管理的文件推送到版本库，打上版本信息

# 07-回滚

git log

 约饭功能上线

commit 1f7b1cd7e1c7c724cd7699be5a8659fa26c2caf4
Author: toprhythm <2573424062@qq.com>
Date:   Sat Feb 12 17:31:25 2022 +0800

    短视频功能上线

commit 32446e7a9302d6ed0afd608712c240212b72ae11
Author: toprhythm <2573424062@qq.com>
Date:   Sat Feb 12 17:29:26 2022 +0800

## 回滚
git reset --hard 32446e7a9302d6ed0afd608712c240212b72ae11
HEAD is now at 32446e7 第一次提交

## 因为回滚玩，再次gitlog只显示前两次的commit，怎么强行回到显示回退前的commit
git reflog
32446e7 (HEAD -> master) HEAD@{0}: reset: moving to 32446e7a9302d6ed0afd608712c240212b72ae11
8df1373 HEAD@{1}: commit: 约饭功能上线
1f7b1cd HEAD@{2}: commit: 短视频功能上线
32446e7 (HEAD -> master) HEAD@{3}: commit (initial): 第一次提交

## 假设回到约饭上线
git reset --hard 8df1373
HEAD is now at 8df1373 约饭功能上线
