# 01_延迟加载js的方式有哪些？

延迟加载：async、defer
<!-- 能获取到本获取不到的dom -->
<script async type="text/javascript" src="script.js"></script>
<script defer type="text/javascript" src="script.js"></script>

defer: 等html解析完成，才会执行js代码,顺次执行的.
async: async是和html解析同步的,不是顺次解析js脚本，谁先加载完，谁先执行

# 03_null和undefined的区别

null是变量被定义了，只是值为null
undefined是变量定义了，但是并未赋值

1. 作者在设计js的时候先设计的null，为什么设计了null？最初作者设计js的时候借鉴了java，有根据c的传统，null自动转0，很不容易发现错误
2. null会被隐式转换成0，很不容易发现错误,因此作者设计了undefined为了填补之前的坑

console.log(typeof null)// 'object'

// 强转为数值，就自动转0
console.log(Number( null ))// 0

// undefined转换数值是为NaN
console.log(Number( undefined ))// NaN

# 05_js微任务和宏任务?

1. js是单线程的语言：同一时间只能做一件事
2. js代码执行流程：同步执行完->事件循环
		
		事件循环【微任务，宏任务】
		
同步的任务都执行完了，才会执行时间循环的内容(click,setTimeout,xhr)
进入事件循环：请求、定时器、事件...
微任务：promise.then...
宏任务: setTimeout...

要执行宏任务的前提是清空了所有的微任务

流程：同步代码执行 -> 事件循环(微任务和宏任务) -> 微任务 -> 宏任务 ...

1. 除了函数外，js是没有块级作用域的

	  注意：如果内部有，优先查找内部，如果内部没有，就查找外部,直到a is not defined
3. 注意声音变量是用var还是没有写（window.b）

  function create ( fn, ...args ) {

    // console.log( args )// [18, '张三']

    // 1. 创建了一个空对象
    var obj = {}// var obj = Object.create({})
    // 2. 将空对象的原型，指向构造函数的原型
    Object.setPrototypeOf(obj, fn.prototype)
    // 3. 将空对象作为构造函数的上下文(改变this指向)
    var result = fn.apply(obj, args)
    // 4. 对构造函数有返回值的判断处理逻辑

  }

    // 前置判断
    if (Array.isArray( obj )) {

    }else {

      }
    }

# 17_localStorage、sessionStorage,cookie的区别

公共点：在客户端存放数据

区别：
1. 数据存放有效期
	sessionStorage ：仅仅在当前浏览器窗口关闭之前有效果【关闭重开浏览器就没了】
	localStorage   : 始终有效，窗口或者浏览器关闭页一直保存，所以叫做持久化存储
	cookie: 只在设置的cookie过期时间之前有效，即使窗口或者浏览器关闭也有效
2. localStorage,sessionStorage不可以设置过期时间
	cookie有过期时间且可以设置过期时间（把时间调整到之前的时间就过期了）
3. 存储大小的限制
	cookie的存储量不能超过4k
	localStorage、sessionStorage不能超过5M
	
	****根据不同浏览器存储的大小是不同的
