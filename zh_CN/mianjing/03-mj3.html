Q: 说一下Vue的生命周期？

A：Vue的生命周期我是这么理解的，就是他从实例化的创建，再到初始化数据，然后再到编译模板，然后Dom渲染，然后数据的更新，以及页面的渲染，然后dom的销毁卸载，从他的生到死的过程，我觉得是这样子的生命周期，然后再生命周期的每个节点，提供了钩子的函数，这样的代码可能有机会被执行，比如说，bc,c,bm,m,bu,u,bd,d,8个周期，如果启用的keep-alive的话，就是10个生命周期，deactivated，activated，第一次进入页面是执行前4个生命周期函数+activated，第二次进入页面只执行activated函数,我对生命周期的理解是这样的， beforeCreated初始化了数据，但是没有挂载到data数据模型，不能拿到data数据，created数据模型已经初始好完成，可以拿到data数据，但是页面的dom，还是获取不到dom的值，在created是可以发送请求，但是这时候不能做页面的dom操作。

Q：做一道promise题: 图片加载失败，用默认图去替代

A:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>
  <script>
    function loadImage(url) {
      var image = new Image()
      return new Promise((resolve, reject) => {
        // input you code
        image.src = url
        document.body.appendChild(image)
        image.onload = function () {
          resolve('load image success')
        }
        image.onerror = function () {
          image.src = 'https://pics5.baidu.com/feed/562c11dfa9ec8a1361a66cf3f7f48b85a1ecc061.jpeg'
          reject('load image error')
        }
      })
    }
    
    loadImage('http://www.baidu.com/不存在.png').then(res => {
      alert(res)
    }).catch((e) => {
      console.log(e)
    })
  </script>
</body>
</html>

Q: 从用户打开网址url，到渲染，他一共完成经历了哪些过程？

A：第一步是先输入网址，然后后端发送到DNS服务器，并且DNS服务器根据域名解析出IP地址，然后第三步与web服务器建立TCP连接，发送浏览器向web服务器发送http请求，然后web服务器响应请求，然后并返回指定的url的数据，或者是错误信息，或者是重定向的一些新的url地址，然后浏览器下载web服务器返回的数据，然后解析html文件，然后生成dom树，然后解析css和js然后渲染页面，然后直到显示结束。

Q：Vue响应式原理,核心思想？

A：当Vue实例创建的时候，Vue它会遍历一下Data选项的那些属性，然后用内部的一个defineProperty方法，然后将他们转化为 getter / setter,然后内部追溯相关依赖，然后在属性被访问和修改的时候通知变化，然后每个组件实例相当于是对应的都有一个watcher的程序实例，然后他会在组件渲染的过程中，把属性记录作为依赖，当依赖的setter被调用，相当于是被改了的时候，他是会通知到wachter，然后重新计算，从而使他相关联的组件进行一个更新，它里面大概这么个情况。

Q：Ajax请求在哪一个生周期（钩子）函数

A：created

Q：data数据存到那个生命周期

A: created因为在这data已经初始化好了，但是beforeCreate的data没有初始话好，不能把数据赋值给数据模型，mounted的时候data初始化好了，dom也初始化好了，但是可能会有一些延迟，所以放到created是最好的。

Q：v-show和v-if的区别？

A: v-show一般控制元素的显示就像css的display: none|block，相当于一个来回切换，但是v-if，他会控制dom的节点存在与否，当我们需要经常切换dom元素隐藏或者显示的时候，我们一般用v-show，然后如果当dom组件只需要一次显示，或者隐藏，一般用v-if比较合理，然后一般我们在权限按钮里用v-if比较多，不能人员显示不同按钮，这个时候就可以选择直接就不需要dom，就会用v-if。

Q：beforeDestory何时会使用？

A: 一般就是在组件销毁前，对页面进行一些特殊的操作，比如说初始化一些函数，然后初始化一些data数据，这时候用的是beforeDestory

Q: delete和Vue.delete()删除这个数组，他们有什么不同之处？

A：delete只是被删除的元素编程undefined，但是其是他的值还是在的，但是Vue.delete是直接删除了相当于是删除了这个数组，相当于一个数逻辑删除，一个是物理删除的区别.

Q：父组件需要和子组件的数据做一个双向绑定，怎么解决？

A：其实就是父组件通过props传值给子组件，然后子组件通过emit通知父组件，然后修改对应的props值，然后相当于是我们比如说用户输入的时候，同步父组件的数据进行一个修改，父组件给子组件传一个props，第二个就是监听子组件的值并同步自己的value属性，然后就实现双向绑定。

Q：说说MVVM理解？

A：MVVM框架一般分为三个，Model、View，ViewModel，model它其实是代表数据模型层，然后数据还有业务逻辑都在model层里面定义，然后view是代表UI视图层负责数据展示，ViewModel，他其实是监听model中的数据改变然后去更新视图的改变，用来处理用户交互，他们之间的关系是，model和view没有直接联系，都是通过viewmodel进行联系的，然后model和viewmodel是有着一个双向数据绑定，就是当model中的数据，进行改变的时候会触发view层的一个刷新，然后view层中，由于一般我们用户操作的时候去改变你们的数据，也会在model中同步，然后这种模式让model和view的数据自动同步，ok

Q: vue中如何监控某个属性值的变化？ 比如监控 data.obj.a: '123'

A：它有两种方式，一种是通过watcher去监控，我们所需要的一个函数的变化，然后进行一些dom操作，然后还有一个属性是comport，然后来监视一个新的，它就是监视同一个数据发生变化。

Q：如何优化，提升网站性能？

A：一般从几个方向：减少http请求的数量，因为浏览器和服务器之间交互需要三次握手，浪费大量时间，而且不同浏览器，对资源请求的数量是有限的，说医用减少http请求数量来优化网站性能，还有一个就是看着资源文件的加载优先级，因为我们有时候其实是触发到某个事件的时候，这样子的话就拍一个优先级顺序，用到的时候在加载，没有到不加载，一般吧css放在头部，把js放在底部，然后还有利用浏览器缓存，就直接将浏览器网络资源缓存到本地，然后下次请求的时候，如果本地资源已经有了，不需要到服务器去请求资源，直接读本地资源，如果没有再去下载远程资源。

Q：什么是重排（重绘）？

A：reflow，他是，相当于类似于dom的变化，影响到了元素的几何属性，比如说盒子的宽和高，然后浏览器会重新计算他的基本属性，渲染上面受到部分的失效，然后浏览器这时候会验证，相当于是dom树上所有的节点，然后visibility属性，如果重排过于频繁，他会使cpu资源消耗上升，所以一般都是减少reflow，如果需要在dom操作去添加样式，一般都是增加css属性，而不是直接去修改state操作样式，ok
