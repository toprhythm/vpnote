# chap03-gitrebase变基和gitflow工作流

# 01-rebase应用场景一

使git记录更简洁

一个大功能要用一周才能完成，第一天提交了一个版本，第二天提交了一个版本...第五天开发完
每天都会提交一次，对于你来说，中间这个提交有意义，但是于老板而言，老板只需要开始的记录，和结束的记录两条即可，rebase可以把多条记录整合为一条记录

echo "print('456')" >> 3.py && git add . && git commit -m "gongnneg:v3"

echo "print('456')" >> 4.py && git add . && git commit -m "gongnneg:v4"

## 法1： 把几条commit记录整合为一条
git rebase -i f43ad43bc20b38fa4ce7e4de4c293a9c71c1f56c(假设这里是v2的commit，那就是包括v2这条记录一直到最新的commit进行一个合并v4, v2->v4合并)

## 把后两条的pick改成 s
pick 8bdf53e gongnneg:v2
s 531be0f gongnneg:v3
s 7b10d4e gongnneg:v4

## Rebase f43ad43..7b10d4e onto f43ad43 (3 commands)
##
## Commands:
## p, pick = use commit
## r, reword = use commit, but edit the commit message
## e, edit = use commit, but stop for amending
## s, squash = use commit, but meld into previous commit
## f, fixup = like "squash", but discard this commit's log message
## x, exec = run command (the rest of the line) using shell
## d, drop = remove commit
##
## These lines can be re-ordered; they are executed from top to bottom.
##
## If you remove a line here THAT COMMIT WILL BE LOST.
##
## However, if you remove everything, the rebase will be aborted.
##
## Note that empty commits are commented out

esc:wq ## 此时会进入提交信息填写

## 可以删除修改为
v2 & v3 & v4 esc:wq


git log

  1. 使用bcompare快速解决冲突
1. 安装beyond compare

2. 在git中进行配置 --local：只对当前项目有效 pro_rebase/
git config --local merge.tool bc3
git config --local mergetool.path '/usr/local/bin/bcomp'
git config --local mergetool.keepBackup false

3. 找到这个安装目录
打开bcompare软件，点击首选项，点击 安装命令行工具
会弹出：
  /usr/local/bin/bcomp:
  启动比较并等待它完成。

  /usr/local/bin/bcompare:
  启动比较并立即返回。

# 04-git命令总结

添加远程地址链接: git remote add 地址.git

推送代码: git push origin dev

下载代码: git clone 地址

拉取代码：git pull origin dev 
					等价于: git fetch origin dev + git merge origin/dev

保持代码提交整洁（变基）: git rebase 分支

记录图形展示,美化gitlog: git log --graph

简洁版本:记录图形展示,美化gitlog: git log --graph --pretty=format:"%h %s"
