# Q: flex布局有哪些常用属性?

  • A：像我们会开启flex布局之后，就是会改变它的一个布局形态，类似于改变它的一个文档流了，平常就用flex-basis,类似于固定它的一个宽和高，还有flex-direction,设置主轴，侧轴啊，像justify-content啊，设置两边或者中间的一种靠拢吧，像flex-grow,定义项目的放大比例啊，默认就是为0，大概就是这么个情况。

# Q：如何实现一个水平垂直居中，用flex？

  • A: display:flex;然后设置justify-content: center;然后设置align-item: center;,就可以实现一个盒子水平垂直居中。

# Q:flex-grow和flex-shrink这两个是什么作用？

  • A: grow的话，是定义项目的放大比例，shrink，定义它的缩小比例，空间不足的话就加缩小

# Q：你如何理解BFC？

  • A:BFC我的理解就是（Block Formatting Context）,块级格式化上下文，就按照字面上去理解，像他的特性的话，就是看作一个隔离功能的容器，它里面的一些布局就不会影响到外部的一些元素，向我们平常使用触发BFC，像浮动啊，然后定位，绝对定位，固定定位，都会触发BFC。

# Q：说说BFC真实使用场景？

  • A: 具体使用场景：像阻止浮动元素的覆盖，然后用于左边的一些定宽，右边自适应，或者是下外边距的一种折叠，可以放在不同的BFC容器中，还有更多的就是清除浮动啊。

# Q：说下JS基本数据类型？

  • A：分为两种：像基本数据类型就是：String，Number，undefined，null以及Symbol，还有Boolean，就是6种，相比复合的就是对象数据类型：对象，数组，函数，都是复合数据类型。

# Q：什么是深浅拷贝？

  • A：其实就是一个对变量进行一个复制的操作吧，像基本数据类型中，就是对他进行一种拷贝，像深拷贝的话，更类似于改变重新获得来的数据的时候不会影响到原来的数据，但是我们理解到的一个像对象这个引用类型，就会把内存的地址也复制给变量，这个时候的改变就会引起原来对象的改变，这个时候就是一种浅拷贝。

# Q：写代码：如何深拷贝？

  • A：几种方法：JSON.stringify JSON.parse 去深拷贝，2.用递归的方式遍历一个对象，让他复制到一个新的对象，然后改变他的key和value，用崭新的对象去代替他，这也是可以实现一个深拷贝。

# Q：说一下Vue2中生命周期有哪些钩子，各自都做了什么事情?

  • A：生命周期理解为像从创建然后到最后销毁组件，像created对应的beforeCreate，创建初始化之前，然后事件数据观测，像created就可以进行数据观测，然后属性方法也会初始化了，可以，但是没有那个$el,像就是说挂载这些，beforeMount,Mounted,beforeMount像render函数被调用，像Mounted，在销毁实例之前，都会把用不到的变量给清空掉，你说的是一些事件绑定是吗

# Q：一般像给后台把异步请求的写在哪里？

  • A：使用Ajax发，然后自己会封装一下，像Vue就用axios，放在created就可以，Vue3没用，但是准备用了，像3.x的话是先导入一个钩子，然后在setup方法中去注册回调，3.x移除了beforeCreate和Created者两个钩子，用setup方法代替的，没跑过vue3

# Q：说一下Vue组件是如何通信的？

  • A：像常见的父子通信啊，非父子之间，然后非父子之间就是我们兄弟组件的通信啊，像父子通信props,自定义事件$emit发布订阅的方式，然后父子之间去监听，还有就是$parent,$children这样去传数据，或者也可以用Provide注入，Inject插入传数据，还有$ref属性,获取组件的属性或者方法，整个组件都可以获取到，还有就是公共的bus，还有就是Vuex的公共的状态管理实现数据共享，状态管理里像state存储数据，actions是异步的，mutations是同步的，还有modules，就是把他给分成一个个模块，然后会借助MapState辅助函数像commit，dispatch这种方式去触发我们的状态

# Q：浏览器从输入URL到页面显示发生了哪些事情？

  • A: 首页是DNS解析，然后建立一些TCP连接啊，然后获取HTML，最后就是浏览器渲染，这是大的流程，页面渲染的流程讲一下：解析它的html，以及css树，根据生成的css树进行回流啊，以及重绘啊，再就是我们所看到的这些页面。

# Q：重绘与回流的区别？

  • A：像回流就是和dom的几何属性是相关的，当dom元素的尺寸发生改变，比如元素宽高，隐藏元素，就要重新计算这个元素的几何属性，然后把他绘制出来，这就是回流。 像重绘的话就是：我们就是说dom元素被修改的但是没有发生一些元素的几何属性的改变，比如只是改变背景颜色，这个时候不需要重新计算几何属性，这个时候称为重绘。

# Q: 重回回流的真实场景，如何优化？

  • A: c3的transform，opacity,像filters这些动画，就是内部引起使得硬件加速，这个时候就是不会引起重绘回流，像will-change
