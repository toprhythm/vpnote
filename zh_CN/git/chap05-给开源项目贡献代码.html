# 01-给开源项目贡献代码

1. 进入https://github.com/
2. 搜索vue,点击fork，整个项目代码被复制到我们的仓库里来
3. 在自己的fork仓库修改open代码
4. git clone 自己仓库的fork库代码下载下来
5. touch src/bugfix.js
6. git add .
7 git commit -m "修复了一个致命的bug"
8. git push origin master
9. 给源代码作者提交 修复bug申请，pull request
10. 项目fork主页，点击new pull request
11. 备注信息，点击贡献按钮
12. 如果他们接受了，他们的源码中就会有你的代码,当然别人也会关闭，拒绝，很正常

# 02-配置文件存放三个位置

## 项目配置文件，仅仅在当前项目生效 项目/.git/config
git config --local user.name 'xxx'
git config --local user.email 'xxx.@xx.com'

## 全局配置文件 
git config --global user.name 'xxx'
git config --global user.email 'xxx.@xx.com'

## 系统配置文件
git config --system user.name 'xxx'
git config --system user.email 'xxx.@xx.com'


## 当前项目
vim ./git/config
---
[core]
        repositoryformatversion = 0
        filemode = true
        bare = false
        logallrefupdates = true
        ignorecase = true
        precomposeunicode = true
[remote "origin"]
        url = git@github.com:toprhythm/star_story.git
        fetch = +refs/heads/*:refs/remotes/origin/*
[branch "master"]
        remote = origin
        merge = refs/heads/master
[branch "dev"]
        remote = origin
        merge = refs/heads/dev
~
---


## 全局 所有的git项目通用

# 05-git做任务管理


issues 话题讨论 （给开源项目提bug）
## 点击项目主页，点issues，输入title，输入desc，勾选左边的assgnees（找一个你要提问的github用户），勾选labels（是bug，还是文档问题，类型）,点击submit new Issues,也可以提bug让这个人去管理

wiki 百科,写wiki，写对项目进行一些介绍
## 点击项目主页，点击wiki，写，这个项目是干嘛的，项目有哪些功能，项目如何运行...,到手一个项目，先看项目wiki有没有写,issues里有没有常用问题，而不是问老大，问同事
