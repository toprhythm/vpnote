# 面试题: Vue2.x生命周期


# 1. 有哪些生命周期
系统自带:

# 创建
beforeCreate
created

# 挂载
beforeMount
mounted

# 更新
beforeUpdate
updated

# 销毁
beforeDestory
destoryed


# 2. 一旦进入到页面或者组件，会执行哪些生命周期，顺序?
# 执行前四个
beforeCreate
created
beforeMount
mounted


# 3. 在哪个阶段会有$el,在哪个阶段会有$data
beforeCreate 啥也没有
created 有data没有el
beforeMount 有data没有el
mounted 都有


# 4. 如果加入了keep-alive会多两个生命周期函数
activated, deactivated # 进入，离开

# 5. 如果加入keep-alive,第一次进入组件会执行哪些生命周期?
beforeCreate
created
beforeMount
mounted
activated

# 6. 如果加入了keep-alive,第二次或者第N次进入组件会执行哪些生命周期?
activated # 只执行一个声明周期,因为vue组件已经被缓存

问：你能说下生命周期？
系统自带了八个:...,第一次进入组件会执行前四个分别是...,如果加入keep-alive，会增加两个生命周期分别为进入和离开，如果第一次进入会执行前五个，前四个+activated，第二次进入只会执行activated

# 面试题：谈谈你对keep-alive的了解

1. ka是什么?
vue系统自带的一个组件，功能：是用来缓存组件的 -> 提升性能

2. 使用的场景?
就是来缓存组件，提升项目的性能,具体实现比如：首页进入到详情页，如果用户在首页每次点击都是相同的，那么详情页就没必要请求N次，直接缓存起来就可以，如果点击的不是同一个那么直接请求即可

computed是有缓存的,methods是没有缓存的

2. computed和watch的区别？

computed可以计算某一个属性的改变，如果某一个值改变了，计算属性会检测到进行返回

# 面试题：props和data谁的优先级更高


github.com/vuejs/vue/blob/dev/src/core/instance/state.js#L48-L62

export function initState (vm: Component) {
  vm._watchers = []
  const opts = vm.$options
  if (opts.props) initProps(vm, opts.props)
  if (opts.methods) initMethods(vm, opts.methods)
  if (opts.data) {
    initData(vm)
  } else {
    observe(vm._data = {}, true /* asRootData */)
  }
  if (opts.computed) initComputed(vm, opts.computed)
  if (opts.watch && opts.watch !== nativeWatch) {
    initWatch(vm, opts.watch)
  }
}

根据源码得出结论

props第一
methods第二
data第三
computed第四
watch第五

# 面试题：打包后运行产生空白页如何解决?


1. 重新创建一个vue项目：使用history模式没有 /#/

2. npm run build,直接找到dist/,直接打开index.html,是空白的,查看源代码,发现<link href="/js/app.fb7d1847.js" />,如果这个路径是./就可以了

3. 创建vue.config.js
module.exports = {
  publicPath: './'
}

4. npm run build ,打开index.html
这时候出来了点东西，Home | About，查看源代码<link href="css/app.9da74d00.css"，确实是相对路径,但是还是显示的不完整

5. 把router里的history改成hash
  mode: 'hash',
  
	hash #
	history
# 再次打包，成功出现页面
npm run build 打开index.html

# 前端如果要测试项目，使用hash

# 面试题：说一下vue的路由模式?


路由模式有两种：history和hash

区别:
	1. 表现形态不同: /#/list ， /list
	2. 跳转请求：
		history: http://localhost:8080/id -> 发送请求
		hash: http://localhost:8080/id -> 不会发送请求
		
	3. 打包后前端自测要使用hash，如果使用history会出现空白页

# 面试题：路由导航守卫有哪些？


全局、路由独享、组件内

1. 全局
beforeEach(to,from,next)、beforeRosolve(to,from,text)、afterEach(to,from,next)

2. 路由独享
beforeEnter(to,from,next)

3. 组件内
beforeRouteEnter、beforeRouteUpdate、beforeRouteLeave


# 理解为保安
# 如果是登录状态，去订单页面，就进去了
# 如果没有登录，去订单页面，强制你先登录
# 你进入到某个页面之前进行拦截，对某一个路由进行拦截，对某一个条件进行拦截

router.beforeEach((to,from,next) => {
	//console.log(to,from,next)
	// to 代表你当前的路由
	// from 你上一个路由
	
	if () {
		next()
	} else {
		router.push('/login')
		next('/login')
	}
	
})

使用场景：判断是否登录，如果登录就next否则就跳转到登录页
